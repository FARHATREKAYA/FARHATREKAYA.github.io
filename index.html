Case Study MVP / Validation Phase

Axiom Fiscal / EL Fatoora Connector

Validation-stage middleware architecture for fiscal compliance workflows in Tunisia's e-invoicing landscape.

Problem

Organizations with legacy upstream systems need a controlled way to satisfy fiscal compliance requirements without destabilizing core financial operations.

Solution

Designed a middleware layer that receives upstream invoice data, enforces compliance rules, and manages submission-state control points.

Architecture

Upstream ingestion adapters, validation engine, artifact generation services, asynchronous submission workflow, failure queue, and persistent audit trail store.

Key Features

CSV ingestion, rule-based validation, compliance artifact generation, submission tracking, failed-invoice diagnostics, and traceable invoice lifecycle history.

Current Status

Private proof of concept in active MVP build. Core workflow orchestration and rule behavior are being validated; production deployment has not started.

My Role

Product and system architecture, backend workflow design, integration boundary definition, and compliance-control implementation.

Why It Matters

Compliance workflows require traceability, clear failure handling, and operational discipline. This project demonstrates that engineering focus in a validation-stage environment.
